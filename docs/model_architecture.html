Img2ImgDiffusionUNet — architecture

Source: src/img2img/model.py · widths shown at model_ch=88 (exp25+ default)

Conventions. Blocks marked [name?] are optional and gated by constructor flags or resolution checks; the constructor leaves them as None when off, which keeps the state_dict clean and permits old checkpoints to load via strict=False.

  • Attention at each level (attn1..attn4) — created only if the level's resolution is in attn_resolutions (e.g. (16, 32, 64) for exp25/exp33). mid_attn at the bottleneck is always present.
  • Temporal attention (tattn*) — created only when use_temporal=True. Inserted after each spatial block at encoder, bottleneck, and decoder; 256px/128px levels are skipped (B×H·W too large for CUDA attention kernels at common batch sizes). Defined in src/img2img/temporal.py.
  • mask_proj — created only when mask_channels > 0 (WAN-style anchor conditioning). Adds a zero-init projection of frame_mask into the stem output x0; identity at init.
  • SourceEncoder + FuseBlock (not drawn; mutually exclusive with source-in-stem) — gated by use_source_encoder. When on, the source is encoded by an ImageNet-pretrained ResNet18 and injected additively at each encoder level via fuse1..fuse4 + mid_fuse. exp25 onward uses --no-source-encoder (source-in-stem); the diagram reflects that default.
  • SourcePyramid + FiLM (exp34, drawn in orange) — created only when use_source_pyramid=True. The pyramid encodes the raw source once per forward into four resolution-matched features f0..f3; each FiLM_dec* module produces (γ, β) from the matching pyramid feature and modulates the decoder activation as x * (1 + γ) + β. Both γ and β are zero-init, so a pyramid-enabled model at step 0 emits the same output as a no-pyramid model with identical backbone weights — making the swap safe to introduce in any existing training recipe.

What the diagram does not show: time-embedding injection (time_mlp feeds every ResBlock via FiLM-style scale/shift on the residual), the optional color_space conversion from linear RGB to sRGB before the source encoder, or per-frame batching in the temporal-attention path (BT = B × T when use_temporal=True).
