Notes

  • Conventions. Boxes marked [name?] are optional and gated by constructor flags or resolution checks; the constructor leaves them as None when off, which keeps the state_dict empty and permits old checkpoints to load via strict=False.
  • Per-level sub-block sequence. Each encoder level is ResBlock (down_i) → [attn_i?] → [tattn_i?]. Each decoder level is cat(↑, h_i) → ResBlock (dec_i) → [attn_dec_i?] → [FiLM_dec_i?] → [tattn_dec_i?]. The bottleneck is mid1 (ResBlock) → mid_attn → [tattn_mid?] → mid2 (ResBlock).
  • Spatial self-attention asymmetry. The original codebase put BottleneckAttention only on the encoder (attn1..attn4, gated by attn_resolutions) and at the bottleneck (mid_attn, always on). The decoder had none. exp34 (--use-decoder-attn) adds the symmetric attn_dec1..attn_dec4 using the same resolution gate as the encoder.
  • Temporal self-attention. tattn* modules (from src/img2img/temporal.py) are created only when use_temporal=True. 256px and 128px levels are skipped because B × H × W exceeds CUDA flash-attention kernel limits at common batch sizes.
  • mask_proj. Created only when mask_channels > 0 (WAN-style anchor conditioning). Adds a zero-init projection of frame_mask to the stem output x0; identity at init.
  • SourceEncoder + FuseBlock (not drawn) — mutually exclusive with source-in-stem; gated by use_source_encoder. exp25 onward uses --no-source-encoder (source-in-stem); the diagram reflects that default.
  • SourcePyramid + FiLM (exp35) — pyramid encodes the raw source once per forward into four resolution-matched features f0..f3; each FiLM_dec* module produces (γ, β) from the matching pyramid feature and modulates the decoder activation as x · (1 + γ) + β. Both γ and β zero-init.
  • t_emb fan-out (not drawn for clarity): every ResBlock (down*, mid*, dec*) takes the time-embedding via FiLM-style scale/shift on its residual.
