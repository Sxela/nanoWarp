# Findings — 2026-05-09 (img2img photo → anime, exp01–exp07)

A working journal of what we learned while running the first round of img2img
experiments on a 287/50 paired photo→anime split (128px, bs=4, 20k-step ceiling).

Findings are ordered by how much they reshaped our mental model of the system,
not chronologically.

## 1. Train-time `x0_hat` panels are misleading by construction

The training loss path computes:

```
t ~ Uniform[0, T)
x_t = sqrt(αbar_t) · target + sqrt(1 - αbar_t) · ε
ε_hat = model(source, x_t, t)
x0_hat = (x_t - sqrt(1 - αbar_t) · ε_hat) / sqrt(αbar_t)
```

`x_t` already contains the answer. At low `t`, `x_t ≈ target`, so `x0_hat`
trivially looks like the target even when the model has not learned the
conditional mapping. Random-`t` mixed panels hide the failure cases (high `t`)
behind the easy cases.

Action taken: rewrote `validate.py` to do **full reverse sampling** for panels
and SSIM/LPIPS, plus a high-`t`-only diagnostic column. That diagnostic is the
honest one — on exp01 it was pure colored static, confirming the model never
learned the source-conditioned reverse trajectory.

## 2. `pix2pix`-style paired data has a structural shortcut

With source ≈ target structurally (same person, same pose, same composition),
the diffusion training objective degenerates. The model can satisfy ε
prediction by approximating `target ≈ source + small_color_shift`, then
computing `ε ≈ (x_t - sqrt(αbar_t) · source_with_color_shift) / sqrt(1-αbar_t)`.

This solves the loss without ever learning the photo→anime structural
transformation. Visible signature: late-step train panels show the predicted
target as "source with a slight palette nudge", not real anime structure.

Action taken:
- added **source dropout** (CFG-style, default 0.15) to break source dependence;
- added **v-parameterization** as a more stable loss target;
- added **high-t warmup** to force the model to confront the hard regime first;
- shipped **direct image-to-image rectified flow matching** as `RectifiedImageFlow`
  in [src/img2img/flow.py](../src/img2img/flow.py). FM eliminates the shortcut
  by removing the noise channel entirely — there is no `x_t` for the model to
  cheat through; the only path between `source` and `target` is the velocity
  field the model is being asked to learn.

## 3. The step-~5000 collapse is the trainable encoder, not the optimizer

Two long runs (exp05 and exp06) both diverged sharply around step 5000–6000,
with the loss curve jumping to a 5–10× higher floor that never recovers.

- exp05: vanilla AdamW, no safety nets → collapsed.
- exp06: + grad clip, LR warmup, cosine decay, source dropout, v-pred, high-t
  warmup → collapsed at the same step.

Optimizer hygiene did not fix it. The smoking gun was the partially-trained
ResNet18 encoder: by default
[`Img2ImgDiffusionUNet`](../src/img2img/model.py#L65-L103) freezes only `stem`
and `layer1`, leaving `layer2/3/4` trainable. Under sustained AdamW updates
with a 287-image dataset, those pretrained semantic features drift
off-distribution and break the conditioning signal the UNet relies on. Once
the encoder representations degrade, x0_hat quality degrades — the visible
"clean reconstructions then suddenly grainy reconstructions" pattern.

Action taken: added `--freeze-source-encoder {none, stem, partial, all}` flag.
The `freeze_stages` mechanism now also calls `.eval()` on frozen modules
(locks BN running stats) and `SourceEncoder.train()` is overridden so frozen
stages survive the recursive train-mode flip the trainer issues each step.

With `--freeze-source-encoder all`: **42.7M total params, 11.2M frozen ResNet,
31.6M trainable in the UNet body / fuses / time MLP / output**. This is the
"addon architecture" test — does a frozen ImageNet encoder + a fresh
trainable UNet produce a reasonable photo→anime model.

## 4. Loss plot autoscale hides the truth

`save_loss_plot` rescales the y-axis to `[min, max]` of all observed losses.
A single big spike (or the step-1 init loss of ~1.2) compresses the early
scatter into the bottom 1% of the chart, making the post-collapse band look
disproportionately elevated. Per-step diffusion loss varies 100× across `t`
anyway — at bs=4 a single batch with 4 high-`t` elements naturally produces a
5× spike that is not a regression.

But the post-collapse band itself is real: the *floor* moves, not just the
spikes. Visual confirmation comes from the panel sequence — `x0_hat` clean
through step 4000, then grainy from step 5000 onward.

A log-scale y axis with a rolling mean would prevent us from falling for the
artifact next time. Not yet implemented; flagged as a follow-up.

## 5. The DDIM `x_next.clamp(0,1)` is mathematically wrong, but masking, not causing, the failure

In [`ddim_step`](../src/img2img/diffusion.py#L88-L100) the intermediate
`x_next` is clamped to `[0,1]`. For a properly normalized diffusion, that is
incorrect: intermediate latents at high `t` are roughly `N(0,1)` and have to
be allowed to leave the `[0,1]` box. `x0_hat` clamping is fine; intermediate
clamping is not.

We tested both: removing the clamp on the under-trained exp01 model produced
high-frequency divergent noise output, while keeping the clamp produced
washed grey scribbles. Neither is good — the *real* problem is the model is
under-trained, the clamp just changes the failure mode.

Reverted the clamp removal so all of exp01–exp06 used the same sampler. We
should revisit this on a properly trained checkpoint and likely remove it
permanently then.

## 6. Inference sampler choice was not the problem

Walked through `--sample-steps` 5 / 20 / 50 (DDIM stride) and 1000 (full
DDPM) on the exp01 baseline. All produced different flavors of failure (noise
/ scribbles / blobs). Neither sampler choice nor step count was the
bottleneck. Sampler quality is downstream of model quality.

## 7. Flow matching is plausibly the right architecture for paired stylization

Direct image-to-image rectified flow matching:

```
x_t = (1 - t) · source + t · target + sigma · noise        for t ∈ [0,1]
v*  = target - source                                       (constant)
sample: x = source; for k steps: x += dt · model(source, x, t)
```

- No noise channel for the model to cheat through, so the structural
  shortcut from finding 2 cannot exist by construction.
- Inference starts at `source` directly, integrating an ODE in 4–10 Euler
  steps — much faster than DDIM-50.
- Same UNet, no architecture change.

Implemented as `RectifiedImageFlow` with the same training_loss/sample
signature as `GaussianImageDiffusion` so the trainer stays method-agnostic.
`--method {diffusion, flow}` flag selects. Checkpoints serialize the method
so validate/infer auto-route.

Early signal from exp07 (flow + `--freeze-source-encoder all` + dropout +
safety nets, 20k step run, in flight at the time of writing): step-2000
panels already show recognizable anime-styled reconstructions in the predicted-target column —
significantly stronger than any diffusion run produced at any step. The hypothesis
that flow + frozen encoder + source dropout breaks the shortcut and stays
stable is being validated in real time.

## 8. The pretrained ResNet's `fc` layer warning is harmless

`SourceEncoder unexpected pretrained keys: ['fc.weight', 'fc.bias']` is
emitted because we load the ImageNet ResNet18 checkpoint into a headless
encoder. We do not need the classifier; the message is informational.
