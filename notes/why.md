# why nanoWarp should exist

There are great "small but real" repos for LLM training.

Visual generation still feels oddly broken:
- tiny notebooks stop before things get useful
- giant frameworks hide the core ideas
- research repos are usually brittle, partial, or overfit to one paper

nanoWarp should let someone:
- understand diffusion from first principles
- train a real image model quickly
- see how video extends image modeling
- modify ideas instead of cargo-culting code
